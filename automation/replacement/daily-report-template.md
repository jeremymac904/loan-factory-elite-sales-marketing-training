### How this report is produced

1. The runner invokes `node scripts/replace-drive-with-youtube.mjs` (DRY-RUN) at 08:00 America/Chicago.
2. The script prints a machine-readable JSON block and a human summary to stdout.
3. The runner pipes stdout into this template (token substitution) and saves a dated copy.
4. On a DRY-RUN with `NEWLY_REPLACED > 0`, the runner notifies Jeremy that a real `--apply` run is ready for approval.
5. Nothing is written to the data file, committed, or pushed without Jeremy's explicit approval.
