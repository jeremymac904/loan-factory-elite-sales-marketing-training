# Audio Drive Upload Plan

**Principle:** Audio files live in Google Drive. GitHub holds metadata only. Local staging holds in-flight files briefly. The website references Drive URLs.

---

## Where audio files live

| Stage | Location | Purpose |
|---|---|---|
| Generated in NotebookLM | NotebookLM Studio | Production sandbox |
| Downloaded locally | `/Volumes/LegendsOS/Jeremy's_2026_Master_Build_Folder/_staging/elite-sales-marketing-notebooklm-audio/` | Temporary holding before Drive upload. Never committed to git. |
| Final hosting | Google Drive: `02 Audio Training / Sales and Marketing 101-601` | Canonical location. Referenced by `audio-companions.json` `driveFileId`. |
| GitHub repo | `audio-companions.json` only | Metadata, not media. |

Audio files larger than 5 MB never enter the repo. Lighter assets (transcripts, small images) can live in the repo if needed.

## File naming convention

| Convention | Example | Rules |
|---|---|---|
| Kebab-case slug | `201-borrower-conversion-audio-companion` | All lowercase, no spaces |
| Core: `<level>-<short-name>-audio-companion.mp3` | `301-referral-partner-growth-audio-companion.mp3` | Level prefix only on core series |
| Bonus: `<slug>.mp3` | `ai-advantage-for-loan-officers.mp3` | No level prefix |
| Transcript: `<same-slug>-transcript.txt` | `201-borrower-conversion-audio-companion-transcript.txt` | Plain text; not auto-generated by NotebookLM, do manually if needed |

No spaces, no parentheses, no version numbers in the filename — that's what Drive's "Manage versions" feature is for.

## Sharing & permissions

| File | Permission | Why |
|---|---|---|
| Audio MP3 | "Anyone with the link can view" | Required so the website can render the audio player without auth |
| Transcript TXT | "Anyone with the link can view" | Same |
| Source PPTX / source docs | "Restricted — Loan Factory team only" | Internal source material, not for public consumption |

The Drive file ID from a sharing URL becomes the `driveFileId` field in `audio-companions.json`. Direct streaming URL pattern:
```
https://drive.google.com/uc?export=download&id=<driveFileId>
```

## Upload checklist (per audio)

For every new audio companion, run this checklist:

```
[ ] 1. Audio generated in NotebookLM Studio using the matching prompt from
       NOTEBOOKLM_AUDIO_PROMPTS_101_601.md or BONUS_SALES_MARKETING_AUDIO_PROMPTS.md
[ ] 2. Audio listened to at 1.5x once for compliance — no rate/APR/fee/approval claim,
       no "MOSO" / "ELO" / "free processing" / "lowest rate" / "guaranteed approval"
[ ] 3. Audio downloaded to /Volumes/LegendsOS/Jeremy's_2026_Master_Build_Folder/_staging/elite-sales-marketing-notebooklm-audio/
[ ] 4. File renamed to the canonical slug (see naming convention above)
[ ] 5. Uploaded to Google Drive under the right subfolder
[ ] 6. Set "Anyone with the link can view"
[ ] 7. driveFileId copied
[ ] 8. audio-companions.json updated with driveFileId, audioFileName, duration, status="downloaded" (or "published" if live)
[ ] 9. AUDIO_PRODUCTION_STATUS_REPORT.md updated
[ ] 10. Local staging file moved to a dated subfolder under _staging so it does not linger in the root
```

## What NOT to do

- Don't commit `.mp3` to GitHub. The `.gitignore` should already exclude `*.mp3` — verify before committing anything.
- Don't put unfinished audio in the public Drive folder. Use `02 Audio Training / Sales and Marketing 101-601 / drafts/` if mid-production.
- Don't store transcripts that include borrower PII anywhere. Strip names and identifiers before saving.
- Don't email audio links to borrowers or partners until the audio has passed the compliance review (Jeremy sign-off).
- Don't reference these files from the website until `audio-companions.json` is updated and the file is public-link.
